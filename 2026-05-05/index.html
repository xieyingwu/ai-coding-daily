xAI Grok 4.3 正式全量发布——百万 Token 上下文 + 永久推理模式 + 60% 降价，对标 GPT-5.5 成本压至 1/10

xAI 于4月30日完成 Grok 4.3 全量推出，这是 Grok 系列迄今最激进的一次价格与能力双重升级：输入价格降约 40%（1.25美元/百万token），输出降约 60%（2.5美元/百万token），同等 benchmark 完整跑测成本仅 395 美元，而 GPT-5.5 同测需 3959 美元——差距10倍。

技术侧升级同样显著：推理能力永久开启（所有查询默认先思考再输出，不再手动切换），上下文扩至 100 万 Token，专业领域表现突出——法律推理（CaseLaw v2，79.3%，排名第一）、企业金融（CorpFin，排名第一）。但通用编码与高难数学（ProofBench 仅11%）仍是短板。同步推出 Custom Voices 语音克隆套件，仅需 120 秒参考音频即可克隆语音，价格低于 ElevenLabs。

对 AI 编程的影响：Grok 4.3 的法律 + 金融专业能力使其成为垂直行业 Agent 的低成本底座，但通用编码能力仍弱于 Claude Code 与 GPT-5.5 体系。

Agent框架 架构演进
Agent 框架工程化成 AI 编程新护城河——运行时设计取代单体模型成核心竞争力

行业共识正在形成：AI 编程助手的竞争重心已从「哪个模型更强」转向「谁的 Agent 运行时框架更好」。Cursor、LangChain、Mistral Workflows 等正竞相落地工程化 Agent 基础设施。

核心趋势包括：① 多智能体通信效率优化——专业化子 Agent（规划/编辑/调试）分工协作，协同编排沉淀在框架而非单个 Agent；② 训练数据质量工程——框架驱动的高质量数据收集超越简单规模扩展；③ 可观测性提升——通过「代码保留率」、用户行为语义分析等指标量化 Agent 输出质量，而非单靠 benchmark。

Cursor 团队博客指出：未来AI辅助软件工程将走向专业化 Agent 分工协作模式，框架工程师的价值将超越提示工程师。

安全警报 供应链攻击
PyPI/npm 包投毒安全警报——Anthropic + Cursor 联合发布安全扫描工具应对 AI 代码供应链攻击

针对 AI 生成代码的供应链安全威胁再度升级：PyPI 和 npm 发现多批投毒包专门针对 AI 生成代码中的常见依赖模式，攻击者利用 AI 工具自动补全倾向于引用「听起来合理的包名」这一特征，定向投毒。

应对动作：Anthropic 推出 Claude Code 内置依赖安全扫描，在代码生成阶段实时检测可疑包引用；Cursor 同步更新安全扫描模块，对 AI 自动补全的包名进行交叉验证。

开发者须知：使用 AI 工具自动生成依赖引用时，务必启用包名验证功能；对于 AI 建议的不熟悉包，在安装前务必在 npm/PyPI 官网核实包的下载量与发布历史。AI 越会写代码，供应链攻击面越大，安全习惯需同步升级。
