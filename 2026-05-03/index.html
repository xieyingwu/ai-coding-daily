GPT-5.5 与 Opus 4.7 双双折戟 ARC-AGI-3：人类100分 顶尖 AI 不足1分，推理鸿沟震惊业界
🔥 头条 AI 局限 基准测试 推理能力

ARC Prize 基金会公布了对 GPT-5.5 和 Claude Opus 4.7 在 ARC-AGI-3 基准测试中的系统性分析——160次运行，两款当前最顶尖的模型合计得分低于1%，而人类首次接触即可100%解决所有问题。

ARC-AGI-3 由 François Chollet 设计，专门测试模型面对「全新逻辑环境」的适应能力——即前所未见的规则与模式，无法从训练数据中记忆。两款模型的失败方式也不同：Opus 4.7 过度自信坚持错误假设，GPT-5.5 提出多种可能性却无法收敛执行。

这一结果引发行业广泛反思：LLM 在可验证任务（代码、数学）上因强化学习能力激增，却在需要真正归纳推理的新场景中束手无策。「刷榜」与「真正推理」之间的鸿沟依然巨大。

来源：ARC Prize 官方博客 · 虎嗅报道

Karpathy 提出「Agentic Engineering」：Vibe Coding 只是抬高地板，真正战场是不降低质量前提下的 10x+ 工程
方法论 Vibe Coding AI 工程

OpenAI 联合创始人 Andrej Karpathy 在 AI Ascent 2026 上分享了对 Vibe Coding 诞生一年的反思。他明确区分两个层次：Vibe Coding 是抬高所有人的能力下限（让非专业人士也能构建软件），而 Agentic Engineering 是在 Vibe Coding 之上建立的更严谨工程学科——核心是保持专业软件的安全与质量标准。

Karpathy 透露，2025年12月出现关键转折：最新模型生成的代码几乎无需手动修正，他对 AI 的信任度质变。他同时警示：「你可以外包你的思考，但永远不能外包你的理解」。Vibe Coding 普及后，最值钱的人类技能是品味、判断力和审慎监督。

来源：腾讯新闻 · Karpathy AI Ascent 2026

Claude Mythos 在 Firefox 148 中发现 271 个安全漏洞，全球监管机构高度警惕 AI 网络安全能力
网络安全 Anthropic Claude Mythos 漏洞挖掘

Anthropic 将 Claude Mythos Preview 向 Mozilla 等少数机构开放测试后，该模型在 Firefox 148 版本中发现了 271 个安全漏洞。Mozilla 表示，所有漏洞已在本周发布的 Firefox 150 中完成修复。

Claude Mythos 不对公众开放的原因之一，正是其近乎完全自主的漏洞挖掘+利用能力：能发现零日漏洞、编写利用代码、串联多个弱点接管主流操作系统和浏览器。美联储、英国央行等金融监管机构已表示正密切追踪该模型的发展动态，担忧其被用于攻击银行基础设施。

来源：腾讯新闻 · Claude Mythos Firefox 漏洞

谷歌豪掷 400 亿投资 Anthropic：Claude Code 获 5GW 算力支撑，AI 编程「两强对垒」格局正式确立
重大资本 谷歌 Alphabet Anthropic Claude Code

谷歌母公司 Alphabet 宣布以 3800 亿美元估值投资 Anthropic 最高 400 亿美元（首期 100 亿立即到账，其余 300 亿待续），同时提供约 5GW 的 TPU 算力资源，专门支持 Claude Code 的持续扩张。

此举被分析人士解读为：谷歌选择「把敌人变成 TPU 最大买家」，彻底终结 OpenAI/Anthropic/Google 三强鼎立的格局，形成 OpenAI vs Anthropic+Google 的双强对垒。AI 编程赛道进入「超级算力」决胜阶段，Claude Code 的未来资源保障大幅增强。

Cursor 持续改进智能体框架：双层评估体系 + 代码保留率指标 + 多模型深度适配，Multi-Agent 架构正式提上日程
Cursor 官方 Agent 框架 评估体系

Cursor 官方发布研究博客《持续改进我们的智能体框架》，首次公开其 Agent 框架核心机制：双层评估体系（离线 CursorBench + 在线 A/B 测试）、代码保留率（统计 AI 生成代码在固定时间后的留存比例，低留存 = 质量差）、以及针对不同模型的深度定制策略。

此前框架以静态上下文+护栏为主，现已转向动态上下文按需获取，随模型能力提升逐步减少限制。关键披露：团队曾用在线实验否决了「高成本模型做上下文摘要」方案（投入产出比太低）。未来规划：向多智能体协同架构演进，专业化分工（规划/快速编辑/调试）是下一阶段核心。

来源：Cursor 官方博客

OpenAI o1 急诊诊断准确率 67% 超越人类医生（50-55%）：AI 编程外，LLM 正在跨界拿下高专业度场景
OpenAI 跨界应用 医疗 AI

新研究结果显示，OpenAI o1 模型结合电子病历与护士提供的简短病情描述，对急诊患者的诊断准确率达67%，而人类分诊医生的准确率仅为50-55%。这是 LLM 在严格对照环境下首次在医疗诊断关键指标上明确超越专业人员的案例之一。

这一结果引发编程界关注：同样的 reasoning 能力，在代码场景（可验证、强化学习训练充分）体现为 SWE-bench 高分，在医疗场景（同样有明确对错标准）也开始出现突破。LLM 的「专业场景跨界」能力边界正在被持续拓展。

来源：LLM Stats News · Techmeme (彭博供稿)

中国法院裁定：企业不得仅以「用 AI 替代员工」为由解雇人员，AI 就业问题进入司法规范阶段
政策监管 AI 就业 中国

中国法院最新裁定：企业不得仅因「使用 AI 替代现有员工」这一单一理由解雇员工。此前（2025年12月）已有另一中国法院作出类似裁决，本次为同类判决的进一步确认。

这是 AI 编程/自动化工具在中国引发就业保障争议后，司法端首次系统性介入的信号。对国内开发团队而言，这意味着以 AI 工具为由的「降本」裁员策略将面临更高法律风险，AI 与人才的协作模式（而非替代模式）将成为合规方向。
