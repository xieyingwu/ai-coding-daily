智能体上下文管理进阶：用「动态上下文」而非「堆文件」让 Agent 跑得更稳
官方研究 稳定性 上下文管理

这是什么？
Cursor 官方最新研究揭示了一个核心反直觉结论：把大量文件预先塞给 Agent，反而会让它变笨、变慢、更容易出错。 官方现在的策略是「动态按需拉取」——只在启动时给 Agent 最基本的上下文（操作系统、git 状态、当前文件），让 Agent 自己去按需读取更多信息。

怎么用？

  • 少塞文件：不要在 prompt 里 @ 大量文件，只 @ 你确定 Agent 当前步骤需要的
  • 让 Agent 自主搜索：描述任务时说清目标而非指路，例如「找到用户认证相关的代码并修复 bug」而非「打开 auth.ts 修改第 42 行」
  • 小步任务分解：把大任务拆成小步骤，每步完成后 git commit，防止上下文积累腐坏
  • 中途切换模型时：Cursor 会自动摘要上下文交接给新模型，但复杂任务尽量全程用同一模型

适合什么场景？
执行多步骤重构、跨文件 bug 修复、长周期 Agent 任务时。按这套方法，工具调用错误率可降一个数量级，任务完成率显著提升。

「代码保留率」自测法：如何判断你用 Cursor 的方式是对的
质量评估 新手必看

这是什么？
Cursor 内部用「代码保留率」评估 Agent 质量——AI 生成的代码，你有多少是直接保留了，多少需要反复修改？这个指标同样适合自我评估你的使用方式是否高效。

自测方法：

  • 如果你经常需要告诉 Cursor「不对，重做」或者自己大改 AI 的输出 → 说明 prompt 不够清晰，需要增加更多约束条件（框架版本、编码规范、目标函数签名等）
  • 如果 Cursor 每次改完都能直接用 → 你已经掌握了高效使用的核心
  • 一个实用技巧：在 .cursorrules 里写入你的项目约束（语言版本、禁用库、代码风格），Agent 每次都会自动遵守，保留率立刻提升
