多代码仓库 Agent 环境——让 Agent 跨 10 个 repo 同时干活

这个功能是什么？
Cursor 新推出的「多代码仓库云端环境」，允许你把企业里所有相关 repo（比如前端、后端、基础设施库）打包成一个 Agent 环境，Agent 可以在所有 repo 之间自由穿梭推理。

怎么用？

  • 进入 cursor.com/dashboard/cloud-agents 创建新环境
  • 添加需要关联的 repo（支持私有仓库），配置好克隆凭证
  • Agent 接到任务后会自动判断涉及哪些 repo，跨仓库发起 PR

适合什么场景？ 微服务架构的团队、前后端联动改动、需要同步更新多个库的基础设施工作。Amplitude 实测：「多仓库支持才让云端 Agent 真正实用起来。」

Dockerfile 配置即代码——把 Agent 运行环境纳入 Git 版本管理

这个功能是什么？
云端 Agent 跑在一个隔离的容器环境里，以前这个环境配置是「隐形的」，出了问题很难排查。现在 Cursor 支持用 Dockerfile 来定义 Agent 的运行环境，这个文件可以提交到 repo 里一起做版本管理。

怎么用？

  • 在环境设置里选择「基于 Dockerfile 配置」，上传你的 Dockerfile
  • 用 --secret 传入私有包注册表凭证（构建完成后自动销毁，不会泄露到运行环境）
  • 层缓存升级后，只有修改过的层才会重建，缓存命中时构建速度提升 70%

适合什么场景？ 团队协作场景（大家共享同一份 Dockerfile）、需要安装私有 npm/pip 包的企业项目、频繁迭代环境配置的场景。不想写 Dockerfile？让 Cursor 帮你生成一份（企业私测中）。

环境版本历史 + 出口网络管控——给云端 Agent 装「安全门」

这个功能是什么？
企业安全的两大需求，Cursor 这次都照顾到了：① 每个环境都有版本历史，出了问题一键回滚；② 可以在环境级别限制出站网络——某个环境只能访问白名单域名，另一个可以开放更多权限，互不干扰。

怎么用？

  • 版本回滚：环境设置 → 版本历史 → 选择时间点回滚（管理员可限制普通用户的回滚权限）
  • 网络管控：为高安全级别环境设置出口白名单，敏感密钥也做到「环境级隔离」，A 环境的 secret 在 B 环境里完全看不到

适合什么场景？ 金融、医疗等合规要求高的企业；需要对 Agent 联网行为做审计的安全团队；多项目并行时防止密钥串用。
