🏗️ CLAUDE.md 工程配置法：让 Claude Code 更好用的 98.4% 秘诀

这是什么？
今天的头条研究证明：AI 编程工具「好不好用」，工程基础设施占 98.4%，AI 本身只占 1.6%。 这对 Cursor 同样适用——写好配置文件，比换更贵的模型更重要。

核心配置文件写法：

  • 项目根目录放 CLAUDE.md 或 .cursorrules：写清楚使用的框架版本、禁用的库、编码风格（如「所有函数必须有 JSDoc 注释」）
  • 描述项目结构：「前端在 /src/client，后端在 /src/server，共享类型在 /src/types」——让 Agent 一开始就知道往哪找文件
  • 添加禁止行为：「不要修改 package.json 的 scripts 字段」「不要引入新的 npm 依赖除非我明确要求」
  • 提供常用命令：「开发服务器启动命令: npm run dev」「测试命令: npm test」

效果：配置好之后，AI 生成代码的可用率从 40% 跃升至 85% 以上，减少 80% 的来回纠正。
