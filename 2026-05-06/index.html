今日 Cursor 技巧
来源：Cursor 官方博客「保持 Cursor 应用稳定」· 2026.04.21
用「工作树隔离」让多个 Agent 同时干活不互相打架 多 Agent
这是什么？ Cursor 官方博客揭秘了他们内部多 Agent 并行时避免冲突的核心方案——Git 工作树（Worktree）隔离。每个 Agent 在独立的工作树里工作，互不干扰文件系统，最后再合并。
怎么用？
  • 用 git worktree add 命令给每个并行任务创建独立目录，让 Cursor 分别在各目录里执行 Agent
  • 比如：同时重构 A 模块 + 修复 B Bug + 更新文档，三个任务三个 worktree，最终 git merge 合入主分支
  • Android 多模块项目尤其适合：不同 SKU lib 的重构任务可以完全隔离并行
适合什么场景？ 有多个互不依赖的任务需要同时推进，或者你想用 Cursor 的背景 Agent 在你工作时同步跑测试/文档更新任务。
检查点（Checkpoint）机制——让 Agent 出错了能「一键回滚」 稳定性
这是什么？ Cursor 的 Agent 模式内置了检查点机制：每次 Agent 执行一个重要操作前会自动保存当前状态快照。如果 Agent 走偏了、改坏了代码，你可以回到任意一个检查点重新开始，而不用手动 git reset。
怎么用？
  • 在 Chat 面板中，Agent 执行记录旁会有「Restore checkpoint」按钮，点击即可回滚到任意历史状态
  • 建议在发出「重构整个模块」这类高风险指令前，先手动做一次 git commit作为保险点
  • 配合 Cursor 的「只读审查模式（Ask Mode）」先预览改动范围，再决定是否执行
核心原则： 大任务 = 小步骤 + 频繁检查点，不要一次让 Agent 改太多文件，每步确认后再继续。
在 Cursor 里接入豆包 Code / DeepSeek V4 的 BYO 模型方案 国内用户
这是什么？ Cursor 支持「BYO Model（Bring Your Own Model）」功能，允许你在 Settings 里配置自定义 API Endpoint，接入任意兼容 OpenAI API 格式的模型——包括豆包 Code、DeepSeek V4 等国内模型。
怎么配置？
  • 打开 Cursor → Settings → Models → Add Model，填入自定义 Base URL 和 API Key
  • 豆包 Code：Base URL 填 https://ark.cn-beijing.volces.com/api/v3，模型 ID 填对应版本
  • DeepSeek V4：Base URL 填 https://api.deepseek.com，模型 ID 填 deepseek-coder-v4
  • 配置好后在 Chat 面板顶部下拉菜单切换即可，两个模型可以随时对比输出质量
适合什么场景？ 网络环境不稳定、想降低 API 成本、或者公司有合规要求必须用国内模型的团队。豆包 Code 每百万 Token 仅 0.6 元，长时间高频调用成本比 Claude 低一个量级。
