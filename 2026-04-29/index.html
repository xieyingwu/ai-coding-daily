⌨️ 今日 Cursor 技巧
Cursor v3.2 多任务革命：/multitask + 工作树 + 多根工作区，一次让 AI 并行处理多个任务

🚀 /multitask：真正的并行处理

在 Cursor v3.2 中，输入 /multitask 后，Cursor 会启动多个异步子智能体同时处理你的请求，而不是排队执行。较大的任务会自动拆分为小块，分配给不同的子 Agent 并行完成。就像从「独自一人写代码」升级为「指挥一个开发小组」。

🌿 工作树：在多个分支间同时推进任务

现在可以在不同 Git 分支上后台并行运行隔离任务，互不干扰。想测试哪个分支的成果，一键切到前台即可。再也不用「完成A功能才能开始B功能」，多条开发线路可以同步推进。

📂 多根工作区：跨仓库一次性修改前后端

单个 Agent 会话现在可以同时操作由多个文件夹组成的工作区（前端仓库+后端仓库+共享库）。修改接口时，前端和后端代码可以在一次对话中同步更新，不用来回切换。适合微服务架构、全栈项目、Monorepo 拆分后的多仓场景。

💡 什么时候用这三个功能？

① 有多个互相独立的功能需要同时开发 → /multitask
② 在多个 Git 分支上并行实验不同方案 → 工作树
③ 需要同时改前端+后端+共享配置 → 多根工作区
三者组合使用，单人也能跑出团队的开发节奏。
