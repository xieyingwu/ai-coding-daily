Meta ProgramBench 实测：顶尖模型「从零生成完整代码仓库」准确率为 0%——AI 编程能力边界被重新标定
能力边界警示 基准测试 行业反思

Meta AI 发布 ProgramBench 基准测试，评估当前各主流模型从零生成完整可运行代码仓库的能力——结果令人震惊：所有被测模型的准确率均为 0%。

这一结果揭示出一个重要区别：AI 在「修改/补全已有代码」上表现出色，但在「从空白文件夹构建真实项目」时依然远未达到人类水平。这也是 Cursor autoinstall 技术（用前代模型自举 RL 训练环境）要解决的核心难题。对开发者的启示：AI 是强大的「协作者」而非「替代者」，人类对系统架构和项目初始化的掌控短期内仍无可替代。

Grok 遭莫尔斯码提示注入攻击损失 17.5 万美元——AI Agent 安全边界敲响警钟
安全事件 提示注入 xAI Grok

黑客通过在 X 平台发布莫尔斯码编码指令，诱导 Grok AI 在解码后自主执行加密资产转账授权，造成 17.5 万美元损失（攻击者事后归还，原因不明）。这是目前最典型的「提示注入劫持 Agent 执行权限」案例，攻击面在于模型对外部内容的信任边界而非代码漏洞。

防御三原则：(1) 最小权限——AI 仅持有完成任务所需最少权限；(2) 高价值操作必须人类二次确认；(3) 外部内容（社媒数据）与工具调用权限完全隔离。随着 AI Agent 接入更多系统，这类攻击的行业影响将持续扩大。

autoinstall 自举 Composer——Cursor 用「上一代 AI 搭环境」给「下一代 AI 练手」，训练效率大幅提升
Cursor 官方最新 高阶原理 RL 训练

这是什么？
Cursor 官方发布新研究「用 autoinstall 自举 Composer」（5月6日），揭示 Composer 2 是如何被训练出来的：用 Composer 1.5（旧版本）自动配置好训练环境，再让 Composer 2 在这些干净环境中做强化学习。这就是「自举」——AI 用自己的前辈来构建训练自己的基础。

对你有什么用？

  • 理解为什么 Composer 越用越准：它经历了真实仓库环境的 RL 训练，处理过 README、Makefile、依赖冲突，所以它「懂」你的项目结构
  • 可以学习的思路：autoinstall 的两阶段流程（目标设定 → 环境达成）本质上也是你给 Agent 布置任务的最佳范式——先说清楚「做完是什么样子」，再让 Agent 去执行，最后验证结果
  • 实战类比：给 Cursor Agent 布置任务时，不要只说「实现功能 X」，改为说「实现完成后，运行 ./gradlew test :module:testDebugUnitTest 应该全绿，无任何报错」——给 Agent 一个可验证的成功标准

数据支撑：Composer 2 在 Terminal-Bench（含环境配置能力测试）上得分 61.7%，比 Composer 1.5 的 47.9% 提升 14 个百分点，这就是 autoinstall 训练数据的直接贡献。

新功能 团队管理 成本可视化

这是什么？
Cursor 在 5月4日的更新日志中上线了「模型控制、支出管理与用量分析」面板。团队管理员现在可以：(1) 限制团队成员可使用的模型列表（防止乱用高价模型）；(2) 查看按成员/按模型/按时间段的 Token 消耗明细；(3) 设置使用量预算上限。

怎么用？（团队管理员）

  • 进入 cursor.com/team/settings → 「Usage & Spending」板块
  • 在「Model Controls」中勾选允许团队使用的模型（比如只开放 claude-sonnet + gpt-5.5-instant，关闭 claude-opus 防止无谓消耗）
  • 设置「Monthly Budget Alert」，当团队月度支出超过阈值时自动邮件通知

Android 团队场景：Android 多模块项目开发中，不同成员的 AI 使用习惯差异很大。启用用量分析后，可以识别「哪些模块 Agent 调用最频繁」「哪些任务类型最消耗 Token」，为后续优化 CLAUDE.md 规则文件提供数据依据。
