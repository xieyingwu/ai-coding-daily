⌨️ 今日 Cursor 技巧

技巧一：用 Cursor 云端 Agent 做「离线批量任务」——出门也能让 AI 帮你写代码

这个功能是什么？
Cursor 云端 Agent 环境允许你在本地不开机的情况下，让 Agent 在云端服务器上运行编程任务，你只需要在手机上看进度、审批操作即可（配合 Codex 移动端思路类似）。

怎么用？

  • 在 Cursor 中配置好 Dockerfile（把你的开发环境依赖写进去），保存后云端就能复现你的本地环境
  • 发起一个需要较长时间运行的 Agent 任务（如批量重构、测试生成），设置好验收条件后提交
  • Agent 在云端环境独立运行，你可以离开电脑做别的事
  • 任务完成后审查 PR，不满意可通过版本历史一键回滚到任意快照

适合什么场景？
批量修改测试用例、跨多个仓库同步 API 接口变更、夜间无人值守的重构任务——任何不需要你盯着的长任务，都适合交给云端 Agent。

技巧二：Google I/O 前夕备战技巧——提前配置 Gemini CLI，第一时间接入 Gemini 4

这个功能是什么？
Google I/O 后天发布，Gemini 4 极大概率上线 Gemini CLI 和 Antigravity IDE，BYO 模型（自带模型）接入是 Cursor 核心能力之一，可在发布第一时间切换体验新模型。

怎么用？

  • 打开 Cursor 设置 → Models → 添加自定义模型（OpenAI 兼容接口）
  • 提前在 Google AI Studio 申请 Gemini 4 API key（I/O 后会开放）
  • 填入 API endpoint 和 key，保存后即可在对话框切换到 Gemini 4
  • 通过「代码保留率」指标对比 Gemini 4 和 GPT-5.5/Claude Opus 4.7 的实际质量

适合什么场景？
喜欢尝鲜最新模型的开发者、想降低 Claude/GPT 成本的团队（Gemini 价格通常更低）、需要超长上下文（200万Token）处理大型代码库的场景。

技巧三：AI 编程「补贴战」实用策略——怎么薅两家巨头的羊毛，同时建立工作流双保险

当前形势：
OpenAI 提供 2 个月免费 Codex，Anthropic Claude Code 额度提升 50%。两家都在争夺你的工作流。

务实建议：

  • 不必二选一：Cursor 支持同时接入 Claude 和 GPT-5.5，用代码保留率衡量哪个模型对你的代码库更有效
  • 把重复性长任务（批量测试、文档生成）交给 Codex Agent（后台跑，免费期间多用）
  • 把需要深度理解代码逻辑的任务留给 Claude Code（额度提升 50%，这阵子多用）
  • 建立自己的 AGENTS.md / CLAUDE.md 工程规则文件，在任何工具之间切换时 AI 都能快速理解项目

核心结论：AI 编程进入「抢习惯」阶段，建立跨工具的项目规则文件，才是真正不被平台绑架的护城河。
